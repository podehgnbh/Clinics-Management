# 🏥 نظام إدارة العيادات الطبية

نظام متكامل وآمن لإدارة العيادات الطبية بشكل إلكتروني كامل، يلغي الحاجة للأوراق ويوفر تجربة سلسة للفريق الطبي والمرضى.

## ✨ المميزات الرئيسية

### 🔐 الأمان والخصوصية
- ✅ نظام **Multi-tenant** متقدم - عزل كامل لبيانات كل عيادة
- ✅ تشفير البيانات الحساسة
- ✅ نظام توثيق آمن عبر البريد الإلكتروني
- ✅ صلاحيات مفصلة حسب الدور الوظيفي

### 👥 إدارة الأدوار
- 👨‍⚕️ **الطبيب**: إنشاء ملفات المرضى، إرسال الطلبات
- 🔬 **فني المختبر**: استقبال طلبات التحاليل، رفع النتائج
- 🩻 **فني الأشعة**: استقبال طلبات الأشعة، رفع التقارير
- 💊 **الصيدلي**: إدارة الوصفات الطبية

### 📋 ملف المريض الإلكتروني
- ✅ بيانات شاملة (الاسم، العمر، الجنس، الأمراض المزمنة)
- ✅ التاريخ المرضي الكامل
- ✅ سجل جميع الطلبات والنتائج
- ✅ إمكانية البحث والفلترة السريعة

### 📨 الإحالات الإلكترونية
- ✅ إرسال طلبات التحاليل مباشرة
- ✅ إرسال طلبات الأشعة مباشرة
- ✅ إرسال الوصفات الطبية مباشرة
- ✅ تتبع حالة الطلب في الوقت الفعلي

### 🔔 الإشعارات الفورية
- ✅ إشعارات داخل التطبيق
- ✅ إشعارات عبر البريد الإلكتروني
- ✅ تنبيهات فورية عند الطلبات الجديدة
- ✅ تنبيهات عند رفع النتائج

### 📁 التخزين الآمن
- ✅ تخزين آمن على S3
- ✅ ربط الملفات بملف المريض
- ✅ إمكانية تحميل الملفات
- ✅ نسخ احتياطية دورية

## 📦 التثبيت والتشغيل

### 1️⃣ استنساخ المستودع
```bash
git clone https://github.com/podehgnbh/Clinics-Management.git
cd Clinics-Management
```

### 2️⃣ تثبيت المكتبات
```bash
pnpm install
```

### 3️⃣ إعداد قاعدة البيانات
```bash
# إنشاء المتغيرات البيئية
cp .env.example .env

# تحديث قاعدة البيانات
pnpm db:push
```

### 4️⃣ تشغيل التطبيق
```bash
# وضع التطوير
pnpm dev

# وضع الإنتاج
pnpm build
pnpm start
```

### 5️⃣ افتح المتصفح
```
http://localhost:3000
```

## 📚 الهيكل الأساسي

```
clinic_management_system/
├── client/                 # واجهة المستخدم (React)
│   ├── src/
│   │   ├── pages/         # صفحات التطبيق
│   │   ├── components/    # المكونات المعاد استخدامها
│   │   └── lib/           # المكتبات والأدوات
│   └── public/            # الملفات الثابتة
├── server/                # خادم التطبيق (Express + tRPC)
│   ├── routers.ts         # الإجراءات (APIs)
│   ├── db.ts              # دوال قاعدة البيانات
│   └── email.ts           # نظام البريد الإلكتروني
├── drizzle/               # قاعدة البيانات
│   └── schema.ts          # تصميم الجداول
└── shared/                # الملفات المشتركة
    └── const.ts           # الثوابت
```

## 🔧 الأوامر المتاحة

```bash
# التطوير
pnpm dev              # تشغيل خادم التطوير
```

## 🔐 متغيرات البيئة

أنشئ ملف `.env` بالمتغيرات التالية:

```env
# قاعدة البيانات
DATABASE_URL=mysql://user:password@host:3306/database

# التوثيق
JWT_SECRET=your-secret-key-here

# OAuth
VITE_APP_ID=your-app-id
OAUTH_SERVER_URL=https://api.manus.im
VITE_OAUTH_PORTAL_URL=https://oauth.manus.im

# معلومات المالك
OWNER_OPEN_ID=owner-id
OWNER_NAME=Owner Name

# APIs الخارجية
BUILT_IN_FORGE_API_URL=https://api.manus.im
BUILT_IN_FORGE_API_KEY=your-api-key
VITE_FRONTEND_FORGE_API_URL=https://api.manus.im
VITE_FRONTEND_FORGE_API_KEY=your-frontend-key
```

## 📖 الاستخدام

### تسجيل عيادة جديدة
1. اذهب إلى الصفحة الرئيسية
2. اضغط "تسجيل عيادة جديدة"
3. أدخل بيانات العيادة
4. تحقق من البريد الإلكتروني

### إضافة موظفين
1. اذهب إلى "تسجيل موظف جديد"
2. أدخل البيانات (الاسم، البريد، الدور)
3. سيتمكن الموظف من تسجيل الدخول

### إنشاء ملف مريض
1. سجل دخول كطبيب
2. اضغط "إضافة مريض جديد"
3. أدخل البيانات الشاملة

### إرسال طلب
1. اختر المريض
2. اختر نوع الطلب (تحليل/أشعة/دواء)
3. أضف الوصف
4. سيتم إرسال إشعار للقسم المختص
